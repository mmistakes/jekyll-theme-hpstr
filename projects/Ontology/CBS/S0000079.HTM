System Architect Symbol details for "Activity State Change"
Activity State (Class)
Description: "Activity" in a state which it is currently or has been previously.
Comment: Note that there are both subtypes (comprising all the possible states) and a "power type" of "Activity State Type". The latter's contents are identical to the subtypes; it exists in order to control topics such as access and permission to amend.
Activity State Change (Class)
Description: "Activity" changing the state of another "Activity".
Agent State Change (Class)
Description:
"State Change Activity" changing the state of an "Agent".


Focus State Change (Class)
Description: "Activity" changing the state of a "Focus For Further Activity".
Intention State Change (Class)
Description: "Activity" changing the state of an "Intention".

State Change Activity (Class)
Description: "Activity" which changes the state of an "Object".
Comment: The "to" state is deemed to take effect simultaneously with the completion of the state change activity. There will of course be occasions when the decision to make a state change is made well in advance of this and that this decision may be communicated to interested parties.
Subject Identifier State Change (Class)
Description: "Activity" changing the state of a "Subject Identifier".
