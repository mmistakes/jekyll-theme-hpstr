System Architect Symbol details for "Not Required Intention"
Abandoned Intention (Class)
Description: "Intention" which has not been fulfilled, but for which there is no further action intended.
Comment: As distinct from "Not Required", where it was never required in the first place.
Agent Accepted Intention (Class)
Description: "Intention" which has been accepted by the originator and/or another "Agent" for action.
Agent Rejected Intention (Class)
Description: "Intention" which has been rejected by the requestee "Agent".
Agent Selected Intention (Class)
Description: "Intention" which has been requested of another "Agent" for action.
Fulfilled Intention (Class)
Description: "Intention" considered to have been completed successfully.
Intention State (Class)
Description: "Intention" in a state which it is currently or has been previously.
Comment: Note that there are both subtypes (comprising all the possible states) and a "power type" of "Intention State Type". The latter's contents are identical to the subtypes; it exists in order to control topics such as access and permission to amend.
Intention Under Consideration (Class)
Description: "Intention" which is appropriate for the "Focus For Further Activity".
Intention/State (Generalization Node)
Base Class: "Intention State"
Definitions Used:
Intention State (Class)
Not Required Intention (Class)
