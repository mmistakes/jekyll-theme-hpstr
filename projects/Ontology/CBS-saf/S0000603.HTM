System Architect Symbol details for "Reason For Abandoning"
Abandoned Activity (Class)
Description: "Activity" which has ended abnormally and for which no further work is intended.
Abandoned Intention (Class)
Description: "Intention" which has not been fulfilled, but for which there is no further action intended.
Comment: As distinct from "Not Required", where it was never required in the first place.
Activity Qualifier Type (Class)
Description: "Descriptive Category" capable of classifying an "Activity Qualifier"
Comment: Necessary to indicate the nature of the qualification as opposed to its value.
Examples: In medication, "Route", "Form", "Strength", "Special Instructions" are all examples of "Activity Qualifier Type".
Activity Type (Class)
Description: "Descriptive Category" capable of being used as a category for "Activities".
Contact Method (Class)
Description: "Category" describing a "Subject Contact".
Comment:
The population of this includes:
Postal address
Telephone number (landline / mobile)
Fax number
Email address
Telex number
Bleeper number.

DC/Describing (Generalization Node)
Base Class: "Descriptive Category"
Definitions Used:
Descriptive Category (Class)
Descriptive Category (Class)
Description: "Category" capable of classifying an "Object" directly rather than by associating it with another "Object".
Comment: As distinct from Associative Category.
Incident Type (Class)
Description: "Descriptive Category" capable of being used as a category for "Incidents".
Intention Type (Class)
Description: "Category" being capable of classifying a purpose of care.
Level (Class)
Description: "Descriptive Category" used to describe the level to which something obtains.
Property Qualifier Type (Class)
Description: "Descriptive Category" capable of classifying a "Classified Property Qualifier".
Comment: Used to indicate how a "Property Qualifier" qualifies a "Subject Property".
Examples: Laterality, site (of the body).

Reason For Abandoning (Class)

Reason For Rejection (Class)
Description: "Category" capable of classifying rejected "Activities", "Foci" and "Intentions".
Comment:
Attribute in CBS v2.

Reason Not Required (Class)
Description: "Category" capable of classifying reasons why "Activities" are not required.
Resource Type (Class)


Description: "Descriptive Category" with the ability to classify "Resource Held".
State Type (Class)
"Descriptive Category" capable of classifying a state of a Class which has a documented state transition.
Subject Characteristic Type (Class)
Description: "Descriptive Category" capable of classifying a "Subject Characteristic".
Comment: Note that Subject Characteristic is the only type of Subject Property which is descriptive, ie not associative.
